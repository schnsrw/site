Casual Sheets

Excel-flavored web spreadsheet

Live

A web spreadsheet that feels like Excel, not Google Sheets — ribbon, formula bar, file-centric flow. Open, edit, and save .xlsx, .ods, .csv, and .tsv with full round-trip fidelity. Real-time co-editing with live cursors, presence avatars, and password-protected rooms. Built on Univer OSS.

  • 180+ end-to-end tests; CI on every push
  • Live co-editing — share dialog, presence avatars, live cursors with name labels, live-typing ghost
  • Worker-side xlsx parse + export; main bundle 6.3 MB; lazy plugin chunks
  • Excel-paste preservation (merges, column widths, row heights); range picker; toolbar overflow
  • Tables with themes, freeze panes, comments, hyperlinks, outline groups, drag-fill
Open source — Apache-2.0 on GitHub. Single-user demo on Pages; co-edit ships with the Docker image. v0.0.4 live on Docker Hub — docker run -p 3000:3000 schnsrw/casual-sheets:0.0.4
Self-host with Docker

Single image. Web + Hocuspocus + Fastify in one container, Redis for room persistence. Anonymous co-edit at /r/<any-id>. 

# Try it
docker run --rm -p 3000:3000 schnsrw/casual-sheets:latest
# open http://localhost:3000

# Persistent + Redis
docker compose up -d

Full setup in the README and the architecture doc.

Casual Editor

Word-flavored web document editor

Live

A web document editor that feels like Word, not Google Docs — ribbon, page layout, file-centric flow. Open a .docx, edit with WYSIWYG fidelity, save it back as .docx. Real-time co-editing via Yjs over a Go gateway; share a link, edit live with anyone. Built on a fork of eigenpal/docx-editor (MIT) — ProseMirror schema with the OOXML model preserved end-to-end.

  • Per-tag round-trip audit on every CI run; current baseline 26/39 fixtures pristine, working to ≥ 90% before the desktop ship
  • Live co-editing — share dialog, presence avatars, remote cursors, loading/disconnected UX
  • Stateless Go backend (own y-websocket gateway) — no DB, no on-disk log; the inline host owns docs while a session is live
  • Three-way fidelity harness in CI: us vs LibreOffice vs OnlyOffice DocumentBuilder, scored per tag
  • Pluggable host integration (inline / WOPI / JWT-API) so the gateway slots into existing storage stacks
Open source — MIT on GitHub. Single-user demo on Pages; bundled co-edit image lands with the first tagged release. Self-host — clone, docker compose up, open http://localhost:8080
Self-host with Docker

Single image. Editor SPA + Go y-websocket gateway in one container. Anonymous co-edit at /?room=<docId> after uploading a .docx via the Share dialog. 

# Build + run locally
git clone https://github.com/schnsrw/docx
cd docx
docker compose up
# open http://localhost:8080

# Dev mode (Vite hot reload + Go run)
docker compose --profile dev up

Full setup in the README and the backend design doc.

Casual Desktop

Offline editor for documents and sheets

Coming soon

Same editors, packaged for the desktop via Tauri — your files stay on your machine, no cloud round-trip, no collab server. A small native binary that opens .docx and .xlsx straight from the OS file dialog and gives you a Word/Excel feel without the install footprint.

  • Single-user only — collab stays on the Docker image
  • Native file dialogs; no localhost, no browser tab
  • Reuses the Casual Editor + Casual Sheets web cores; deltas live in the Tauri shell
  • Targets macOS, Linux, and Windows from one codebase
Status — early Tauri scaffolding. First binary lands after the docs & sheets web fidelity hits its 90% floor.
