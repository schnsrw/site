Overview

Casual Sheets is a web spreadsheet that feels like Excel, not Google Sheets — ribbon, formula bar, file-centric flow. Open, edit, and save .xlsx, .ods, .csv, and .tsv with full round-trip fidelity. Real-time co-editing with live cursors, presence avatars, and password-protected rooms — gated to the Docker image so the hosted Pages demo stays single-user.

Built on Univer OSS (Apache-2.0); Casual Sheets adds the office-style ribbon, file flow, co-editing transport, share UX, and self-host packaging.

Status — Phase 2.2 shipped. 180+ Playwright tests pin the editor, live co-edit with cursors / presence / live-typing ghost runs in the Docker image, large-file pipeline moved xlsx parse + export off the main thread.

Features

Editing

  • Office-style ribbon — Home, Insert, Formulas, Data, View, Review.
  • Inline cell editing, F2, Backspace, Delete, Escape; formula bar with editable Name Box (type B5 to jump).
  • Fonts, colors, fill, wrap, alignment, borders (split-button dropdown with 7 modes + color picker), cell merge / unmerge.
  • Tables with themes, freeze panes, comments, hyperlinks, outline groups, drag-fill (Ctrl+D / Ctrl+R), relative-ref formula extension.
  • Sort, multi-column sort dialog, filter, AutoSum / Average / Count / Min / Max.
  • Open / save .xlsx / .ods / .csv / .tsv with workerized parse + export; drag-and-drop file open; save / export toasts.
  • Excel-paste preservation — merges, column widths, row heights survive a copy-paste from Excel.

Collaboration (Docker image)

  • Live co-editing via Yjs + Hocuspocus + Redis persistence.
  • Share dialog, password-protected rooms, presence avatars, live cursors with name labels, live-typing ghost overlay.
  • Anonymous co-edit at /r/<roomId>.

Quality & ops

  • 180+ end-to-end tests; CI on every push.
  • Worker-side xlsx parse + export; main bundle 6.3 MB; lazy plugin chunks.
  • Loading overlay with phase-aware progress for large workbooks.

Architecture

Browser
  ├─ Univer editor — ribbon, formula bar, sheets surface
  ├─ Yjs adapter (op log into Univer's command system)
  └─ y-websocket / WebSocket transport
                  │
                  ▼
Single Docker container
  ├─ Fastify HTTP server (static SPA + REST)
  ├─ Hocuspocus WebSocket server
  │    persists per-room state to Redis
  └─ Worker pool — xlsx parse / export, off the request path
                  │
                  ▼
Redis (sidecar)
  per-room snapshots + presence

The single-user Pages demo skips Hocuspocus and Redis entirely — the bundle is the same, just built with VITE_COLLAB_ENABLED=0.

Configuration

Environment variables for the Docker image:

PORT
HTTP port the bundled server listens on. Default: 3000.
REDIS_URL
Redis connection string for room persistence. Default: redis://redis:6379 (the sidecar in docker-compose.yml).
VITE_COLLAB_ENABLED
1 to expose the share dialog and connect to Hocuspocus on the same origin. The Pages demo builds with this off.
HOCUSPOCUS_PATH
WebSocket path the Hocuspocus server is mounted on inside the bundled image. Default: /collab.
MAX_UPLOAD_MB
Upper bound on the workbook size the worker pool will accept. Default: 100.

FAQ

Why Univer instead of building a sheet engine from scratch?
A correct, performant sheet engine — formula evaluation, dependency tracking, range arithmetic, xlsx round-trip — is years of work. Univer ships that under Apache-2.0; Casual Sheets focuses on the Office-style flow on top.
Why a Docker image instead of a Vercel deployment for the collab story?
Hocuspocus needs a long-running WebSocket process, and Redis as a sidecar. That's not a Vercel function. A single image with Fastify + Hocuspocus + Redis is the simplest self-host shape that actually persists.
Will rooms persist after everyone leaves?
Yes — Redis holds the per-room snapshot. The Hocuspocus server flushes on update; rooms can be rehydrated days later.
Does the hosted demo support co-editing?
No — Pages can't run a WebSocket server. Co-edit lives in the Docker image; sheet.schnsrw.live is the single-user feature demo.
